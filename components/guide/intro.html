介绍 #

前言 #

在接连推进穗学车、穗托育、穗培训三个项目的过程中，我渐渐发现了一个共性：这些业务虽各有侧重，核心却都围绕资金托管场景展开，从流程逻辑到UI设计更是高度相似。

每次开发时，即便业务逻辑相同、UI 设计规范统一，那些样式和交互完全一致的表单组件、交互模块，还是得在不同项目里重复编写；就连各类通用性极强的公共方法，也逃不过在三个项目中反复复制代码的命运。这种重复劳动使得后期迭代时，哪怕只是调整一个方法逻辑，都要在三个项目里逐一操作修改，不仅耗费大量精力，还很容易出现漏改、各项目版本不一致的问题。

正是这些重复劳动带来的困扰，让我萌生了搭建公司内部团队 UI 组件库的想法 —— 把三个项目中通用的资金托管场景组件、标准化的 UI 元素沉淀下来，更要将各类公共方法统一封装。通过内网仓库集中管理后，既能让各项目直接复用成熟组件，也能通过统一的方法调用避免重复编码，实现 “一处修改、全项目同步”。

由来 #

而这个组件库之所以命名为ParametersUI，正源于我们“参数科技”的公司名称——既呼应了企业标识，也暗含“用标准化参数定义组件，让开发更高效”的设计理念。为了区分于第三方组件，所有组件都采用P作为前缀（如p-table、p-confirm），全局公共方法则统一挂载在$p对象下（如this.$p.validatePhone()），既便于团队快速识别内部组件，也让调用逻辑更具一致性，从命名到使用都形成了鲜明的“参数特色”。

这便是ParametersUI组件库的由来：源于实际业务中的痛点，依托公司品牌基因而生，让团队从机械的重复工作中脱身，更专注于业务本身的创新与优化。

安装 #

在项目根目录执行如下命令即可：

npm i parameters-ui --save

全局引入 #

js
// main.js
import ParametersUI from 'parameters-ui'
Vue.use(ParametersUI)
js
// pages.json
 "easycom": {
    "autoscan": true,
    "custom": {
        "^p-(.*)": "parameters-ui/components/p-$1/p-$1.vue"
    }
}

按需引入 #

js
// 非小程序端
#ifdef !MP
import { pContract } from 'parameters-ui/components';
Vue.component(pContract.name, pContract)
#endif
js
// 小程序端
import pContract from 'parameters-ui/components/p-contract/p-contract.vue';
export default {
    components: {
        pContract
    }
}

组件使用 #

html
<p-contract @closePopup="getSignature" />
